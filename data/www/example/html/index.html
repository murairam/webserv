File upload

Uploads are served from the /upload/ location (alias ./data/www/example/uploads/). The form below talks directly to that endpoint using multipart/form-data.

Upload responses will appear here.

Browse uploads: visit /upload/ to see the autoindex and download stored files.

Rename or replace: use the PUT helper below or run curl -T myfile.txt http://localhost:32000/upload/custom-name.txt.

HTTP method playground

These forms and helpers talk to the server so you can verify routing, body parsing, and upload/overwrite behaviour for the mandated methods.

GET query tester

Submit the form to append a query string and observe CGI variables.

POST form data

Posts form-encoded data to the PHP CGI so you can verify request body parsing.

PUT helper

Stores or replaces a file in /upload/. Adjust the filename or body as needed.

DELETE helper

Remove a file from /upload/. Use it after PUT uploads to verify clean-up.

Method responses will appear here.
